מוניטור מכווני טיסה - Flight Instruments Monitor

פרויקט Full-Stack (ריאקט, נוד ומונגו) המציג מערכת לניהול ומעקב בזמן אמת אחר נתוני טיסה בקוקפיט: מד גובה (Altitude), מצפן (HSI) ואופק מלאכותי (ADI).

הפרויקט מבוסס על קוד נקי ומודולרי, נכתב ב-TypeScript בצד הלקוח ומעוצב באמצעות CSS מותאם אישית ללא שימוש בספריות עיצוב חיצוניות, תוך שמירה על ביצועים גבוהים ואנימציות תנועה חלקות. המערכת פרוסה (Deployed) בארכיטקטורת קונטיינרים (Docker) מאחורי שרת Reverse Proxy.

קישור למערכת באוויר (Production):
https://central-proxy.bluebuck-tailor.ts.net/jacobFlightMonitor/

תכונות מרכזיות במערכת

תצוגה ויזואלית (Visual Mode)

מד גובה (Altitude): סקאלה אנכית בטווח של 0 עד 3000 רגל. קו החיווי הלבן זז בצורה חלקה ב-CSS בהתאם לגובה המדווח על ידי המרת הערך לאחוזים בזמן אמת.

מצפן (HSI): דיסק עגול המסתובב נגד כיוון השעון בהתאם למעלות (0 - 360 מעלות) באמצעות תכונת ה-rotate ב-CSS, כאשר חץ המטוס במרכז נשאר קבוע ויציב ומצביע תמיד כלפי מעלה.

אופק מלאכותי (ADI): עיגול חצוי שמשנה את צבע הרקע שלו בצורה חדה ומדויקת בין כחול (שמיים) לירוק (אדמה) על ידי שימוש ב-linear-gradient המקבל את אחוז הזווית המדווח (-100 - 100).

תצוגת נתונים יבשים (Text Mode)

על פי דרישות הממשק העדכניות, המערכת מציגה את הנתונים הטקסטואליים בתוך שלוש כמוסות (אליפסות/קפסולות) מעוגלות בשורה אופקית אחת, הממוקמות בדיוק במקום שבו מופיעים המכשירים הגרפיים. עיצוב זה שומר על שפה עיצובית אחידה (UI Consistency) ומונע קפיצות של התוכן בעת מעבר בין המצבים.

חלון הזנת נתונים (Data Dialog)

לחיצה על כפתור ה-+ הירוק פותחת חלון צף (Modal) להזנת נתוני טיסה חדשים. המערכת מיישמת וולידציה כפולה (Full-Stack Validation): הנתונים נבדקים ונחסמים מפני חריגות מהטווח המותר גם בצד הלקוח (לפני שליחת הבקשה) וגם בצד השרת (בתוך הסכמה של Mongoose), מה שמבטיח בסיס נתונים תקין ומאובטח.

טעינה וסנכרון נתונים

ברגע העלייה של האפליקציה, היא פונה אוטומטית לשרת בבקשת GET, שולפת את הנתון העדכני ביותר שנשמר ב-MongoDB (באמצעות מיון תאריכי יצירה הפוך במאפיין createdAt) ומעדכנת מיד את המכשירים על המסך ללא צורך בריענון ידני.

ארכיטקטורת רשת ופריסה (Deployment Architecture)

המערכת רצה בתוך סביבת ייצור מבודדת (LXC Container ב-Proxmox) בכתובת פנימית 10.0.0.5, ומונגשת החוצה באמצעות רשת Tailscale.

Reverse Proxy: שרת Nginx מרכזי מקבל את הבקשות מכתובת ה-Tailscale Funnel ומנתב אותן פנימה.

Subpath Routing: המערכת מוגדרת לפעול תחת הנתיב /jacobFlightMonitor/. כל הבקשות (קבצים סטטיים וקריאות API) מטופלות בהתאם לנתיב זה.

Docker Compose: האפליקציה מחולקת לשלושה שירותים (Services) נפרדים המתקשרים ביניהם ברשת וירטואלית פנימית:

flight_client: שרת Nginx פנימי המגיש את קובצי ה-React הבנויים.

flight_server: שרת Node.js / Express המטפל בלוגיקה העסקית.

mongo: מסד נתונים MongoDB.

שדרוג תשתיות ואוטומציה (DevOps & CI/CD)

כחלק המרכזי במעבר מסביבת פיתוח מקומית לסביבת ייצור (Production) מנוהלת על גבי שרת LXC ב-Proxmox, הוגדרו ונבנו המרכיבים הבאים:

אוטומציית פריסה (CI/CD עם GitHub Actions):
הקמנו מנגנון פריסה אוטומטי המאפשר לעדכן את השרת החי ישירות מתוך סביבת הפיתוח המקומית, ללא צורך בהתחברות ידנית לשרת.

התקנת סוכן מקומי (Self-Hosted Runner) על גבי שרת ה-LXC הרץ ברקע כשירות קבוע.

יצירת קובץ ניהול תהליכים של GitHub Actions המאזין לדחיפות (Push) לענף הראשי ומריץ אוטומטית פקודות משיכה ובנייה מחדש.

הוראות התקנה והרצה (באמצעות Docker)

דרישות קדם:

שרת או מחשב עם Docker ו-Docker Compose מותקנים.

שלב 1: הגדרת משתני הסביבה (קריטי להרצה)

כנס לתיקיית ה-server וצור את קובץ ה-.env. וודא שהגדרות החיבור למונגו (MONGO_URI) מוגדרות כך שיפנו לשירות ה-mongo ב-Docker Compose.

שלב 2: בנייה והרצת המערכת

פתח טרמינל בתיקיית השורש של הפרויקט והרץ:

docker-compose up --build -d


המערכת תעלה את כל השירותים ברקע.

ניהול גרסאות והיסטוריית פיתוח

כדי להבטיח מעקב מדויק אחר התפתחות הפרויקט, פיצלנו את היסטוריית הפיתוח ב-Git:

הענף offline-local-version והתגית v1.0-offline מסמנים את נקודת הסיום של גרסת הפיתוח המקומית (לפני המעבר לארכיטקטורת Docker ו-Proxmox).

נקודות מפתח להערכת הפרויקט

כתיבה ותיעוד הקוד: כל קבצי הקוד מתועדים בצורה מלאה שורה אחר שורה בעברית ברורה.

שימוש מנוסה ב-CSS3: בניית המכוונים נעשתה ללא שימוש בספריות עיצוב חיצוניות או קנבס, אלא על ידי שימוש מתקדם ב-transition, transform ו-linear-gradient.

יציבות המערכת ופתרון תקלות: הקוד עטוף בבלוקים של try/catch, ופתרונות ברמת השרת מבטיחים יציבות בסביבת הייצור.
