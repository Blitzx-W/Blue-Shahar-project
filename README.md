מוניטור מכווני טיסה - Flight Instruments Monitor

פרויקט Full-Stack (ריאקט, נוד ומונגו) המציג מערכת לניהול ומעקב בזמן אמת אחר נתוני טיסה בקוקפיט: מד גובה (Altitude), מצפן (HIS) ואופק מלאכותי (ADI).

הפרויקט מבוסס על קוד נקי ומודולרי, נכתב ב-TypeScript בצד הלקוח ומעוצב באמצעות CSS מותאם אישית ללא שימוש בספריות עיצוב חיצוניות, תוך שמירה על ביצועים גבוהים ואנימציות תנועה חלקות.

תכונות מרכזיות במערכת

1. תצוגה ויזואלית (Visual Mode)

מד גובה (Altitude): סקאלה אנכית בטווח של 0 עד 3000 רגל. קו החיווי הלבן זז בצורה חלקה ב-CSS בהתאם לגובה המדווח על ידי המרת הערך לאחוזים בזמן אמת.

מצפן (HIS): דיסק עגול המסתובב נגד כיוון השעון בהתאם למעלות (0 - 360 מעלות) באמצעות תכונת ה-rotate ב-CSS, כאשר חץ המטוס במרכז נשאר קבוע ויציב ומצביע תמיד כלפי מעלה.

אופק מלאכותי (ADI): עיגול חצוי שמשנה את צבע הרקע שלו בצורה חדה ומדויקת בין כחול (שמיים) לירוק (אדמה) על ידי שימוש ב-linear-gradient המקבל את אחוז הזווית המדווח (-100 - 100).

2. תצוגת נתונים יבשים (Text Mode)

על פי דרישות הממשק העדכניות, המערכת מציגה את הנתונים הטקסטואליים בתוך שלוש כמוסות (אליפסות/קפסולות) מעוגלות בשורה אופקית אחת, הממוקמות בדיוק במקום שבו מופיעים המכשירים הגרפיים. עיצוב זה שומר על שפה עיצובית אחידה (UI Consistency) ומונע קפיצות של התוכן בעת מעבר בין המצבים.

3. חלון הזנת נתונים (Data Dialog)

לחיצה על כפתור ה-+ הירוק פותחת חלון צף (Modal) להזנת נתוני טיסה חדשים.
המערכת מיישמת וולידציה כפולה (Full-Stack Validation): הנתונים נבדקים ונחסמים מפני חריגות מהטווח המותר גם בצד הלקוח (לפני שליחת הבקשה) וגם בצד השרת (בתוך הסכמה של Mongoose), מה שמבטיח בסיס נתונים תקין ומאובטח.

4. טעינה וסנכרון נתונים

ברגע העלייה של האפליקציה, היא פונה אוטומטית לשרת בבקשת GET, שולפת את הנתון העדכני ביותר שנשמר ב-MongoDB (באמצעות מיון תאריכי יצירה הפוך במאפיין createdAt) ומעדכנת מיד את המכשירים על המסך ללא צורך בריענון ידני.

הוראות התקנה והרצה

דרישות קדם

התקנת Node.js (גרסת LTS מומלצת).

שרת MongoDB פעיל שרץ ברקע (על פורט ברירת מחדל 27017).

שלב 1: הגדרת משתני הסביבה (קריטי להרצה)

כדי שהשרת יוכל להתחבר בהצלחה לבסיס הנתונים ולהאזין בפורט הנכון, עליך להגדיר את קובץ הגדרות הסביבה שלו:

כנס לתיקיית ה-server.

אתר בתוכה את הקובץ המוכן מראש שנקרא env.example.

העתק את הקובץ env.example, הדבק אותו בתוך תיקיית ה-server (או בתיקיית ההגדרות של השרת) ושנה את שמו של הקובץ החדש ל-.env בלבד.

פתח את הקובץ שנוצר וודא שההגדרות מתאימות לסביבה שלך (כמו הגדרת ה-PORT או ה-MONGO_URI).

שלב 2: התקנה והרצת השרת (Backend)

פתח טרמינל ונווט לתיקיית השרת:

cd server


התקן את החבילות הנדרשות (Express, Mongoose, Cors, Dotenv):

npm install


הפעל את השרת:

npm start


לאחר ההפעלה, תראה בטרמינל הודעות המאשרות שהשרת מאזין בפורט 5000 ושהחיבור ל-MongoDB עבר בהצלחה.

שלב 3: התקנה והרצת ממשק המשתמש (Frontend)

פתח חלון טרמינל נוסף ונווט לתיקיית הלקוח:

cd client


התקן את חבילות הקוד של ה-React:

npm install


הפעל את שרת הפיתוח המקומי (מבוסס Vite):

npm run dev


כנס לקישור המקומי שמופיע בטרמינל (בדרך כלל http://localhost:5173) כדי לראות את המערכת פועלת בדפדפן.

נקודות מפתח להערכת הפרויקט

כתיבה ותיעוד הקוד: כל קבצי הקוד (רכיבי הריאקט, קוד השרת ומבנה הנתונים) מתועדים בצורה מלאה שורה אחר שורה בעברית ברורה, המפרטת את הלוגיקה והמניעים מאחורי כל פקודה (בהתאם לסעיף 1.2 במסמך הדרישות).

שימוש מנוסה ב-CSS3: בניית המכוונים נעשתה ללא שימוש בספריות עיצוב חיצוניות או קנבס, אלא על ידי שימוש מתקדם ב-transition, transform ו-linear-gradient כדי לייצר תנועה חלקה בזמן עדכון הסטייט.

יציבות המערכת וטיפול בשגיאות: הקוד עטוף בבלוקים של try/catch בצד השרת ובצד הלקוח, מה שמבטיח שאם תתרחש שגיאת תקשורת המשתמש יקבל חיווי מסודר על המסך והאפליקציה לא תקרוס.
